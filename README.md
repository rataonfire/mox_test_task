# 🐇 Ферма невидимых кроликов

## Что это

Интерактивный дашборд для оценки числа кроликов по косвенным сигналам — движение датчиков, следы, ямки, шорохи. Приложение рассчитывает оценку в реальном времени, показывает уверенность расчёта, объясняет каждую цифру и даёт рекомендации фермеру.

🔗 Прод: [URL будет добавлено при деплое]

## Почему этот сценарий

**Выбран Вариант 02 — «Ферма невидимых кроликов».** Это богатая, но компактная предметная область: детерминированная объяснимая модель (можно запечь в код и проверить вручную) плюс живая интерактивность (всё пересчитывается на лету при каждом нажатии слайдера).

## Как запустить локально

```bash
npm install
npm run dev
npm test
```

Опционально: `npm run build` для сборки продакшена.

## Стек и структура

React 19 + Vite + TypeScript (strict mode) с самописным CSS. Движок оценки (src/engine/) — чистые функции без React-зависимостей, легко тестируются. Компоненты UI (src/components/) содержат только вывод готовых вычислений, без модельной математики. Сценарии и журнал AI-разработки в src/data/. Тестирование — Vitest (35 тестов), включая закреплённый ручной пересчёт эталонных данных. Smoke-тест для приёмки — e2e/smoke.spec.md.

## Как устроена модель оценки

### Формула вклада

Каждый сигнал вносит вклад:
```
вклад = count × вес_типа × (0.5 + интенсивность/10)
```

Веса типов сигналов (по умолчанию):
| Тип | Вес | Обоснование |
|-----|-----|---|
| new_hole (ямка) | 1.2 | Ямки почти всегда роют кролики — сильный прямой след |
| motion_sensor (датчик движения) | 1.0 | Датчик надёжен, но срабатывание = минимум один кролик |
| rustle_detected (шорох) | 0.5 | Шорох слышен, но источник неоднозначен |
| missing_carrot (морковь пропала) | 0.4 | Один кролик съедает несколько морковок |
| footprints (следы) | 0.3 | Следы легко пересчитать — один кролик топчет много |

### Агрегация по локациям

Сигналы группируются по локации (нормализация: без учёта регистра, обрезка пробелов). Внутри каждой локации вклады суммируются со скидкой за пересечение:
```
оценка_локации = c₁ + c₂·d + c₃·d² + ...
где d = 0.6 (скидка за пересечение)
```

**Пример (Сарай):** вклады [1.5, 1.3] → 1.5 + 1.3×0.6 = 2.28 кроликов

### Итоговая оценка

Суммируются оценки всех локаций **без округления**, затем округляется итог:
```
оценка = max(round(Σ оценок_по_локациям), 1) если есть активные сигналы, иначе 0
```

### Уверенность расчёта

Комбинированная оценка (0–100%) по четырём факторам:
- **Разнообразие (30%):** сколько разных типов сигналов (max 5)
- **Перекрёстность (30%):** сколько локаций с ≥2 сигналами
- **Объём (25%):** общее число сигналов (норма 8)
- **Чёткость (15%):** средняя интенсивность сигналов

Ярлык:
- < 40% → «низкая» (красный)
- 40–70% → «средняя» (жёлтый)
- > 70% → «высокая» (зелёный)

### Вилка неопределённости

```
ширина = (1 − уверенность/100) × 0.5
низ = round_down(оценка × (1 − ширина))
верх = round_up(оценка × (1 + ширина))
гарантия: диапазон не уже ±1
```

### Пример пересчёта исходных данных

**Исходные сигналы:**
- evt_001: 5 морковок, интенсивность 4, Огород → 5 × 0.4 × 0.9 = 1.8
- evt_002: 2 ямки, интенсивность 7, У забора → 2 × 1.2 × 1.2 = 2.88
- evt_003: 1 датчик, интенсивность 8, Сарай → 1 × 1.0 × 1.3 = 1.3
- evt_004: 3 шороха, интенсивность 5, Сарай → 3 × 0.5 × 1.0 = 1.5
- evt_005: 6 следов, интенсивность 6, Теплица → 6 × 0.3 × 1.1 = 1.98

**По локациям:**
- Огород: 1.8
- У забора: 2.88
- Сарай: 1.5 + 1.3 × 0.6 = 2.28
- Теплица: 1.98

**Итого:** 1.8 + 2.88 + 2.28 + 1.98 = 8.94 → **≈9 кроликов**

**Уверенность (взвешенная сумма):** 30×1.0 (разнообразие 5/5) + 30×0.25 (перекрёстность 1/4) + 25×0.625 (объём 5/8) + 15×0.6 (чёткость 6/10) = 62.1 → **62% (средняя)**

**Диапазон:** 9 × (1 ± 0.19) = **7–11 кроликов**

*Эти числа закреплены в тестах (src/engine/estimate.test.ts).*

## AI-инструменты и проверка результата

**Процесс разработки:** Claude Code работал как оркестратор; каждый этап вели специализированные субагенты (архитектор → имплементеры по слайсам → два независимых адверсариальных ревьюера → критик UX → верификатор). Между фазами — человеческие гейты (молчание = согласие); все ключевые решения записаны в WORKLOG.md.

**Проверка результата:** 35 тестов Vitest, включая ручной пересчёт исходных данных (8.94 → 9, 62%, 7–11) — закреплён в коде. На каждом шаге: `npm run build` (exit 0). Живая интерактивная проверка в браузере (скриншоты 1280×800 и 390×812 пикселей, никакого горизонтального переполнения). Адверсариальное ревью Phase 3 нашло и исправило реальные баги: горизонтальный скролл на узких экранах, подмену объяснения в hero-карте (0.62 маскировалась под 0.6), жаргон в русской ошибке импорта.
