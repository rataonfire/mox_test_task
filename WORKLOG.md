## CP-1 — Архитектурный план и первые ошибки AI (Phase 1)
Задача, которую я поставил AI (агент-архитектор, чистый контекст): составить PLAN.md по спецификации §2–§4 — дерево компонентов, модель состояния, замороженный API движка, 3 слайса сборки, таблица edge-cases, out-of-scope. Кода не писать.
Что предложил AI: полный PLAN.md с 8 флагами [DECISION], 17 edge-cases, 10 smoke-тестами.
Моё решение: принял план со **тремя исправлениями** (ошибки архитектора, найденные ручной сверкой со спецификацией):
1. **[DECISION #2] отклонён** — архитектор предлагал «реверс-инжинирить» константу uncertaintyWidth, хотя формула из ТЗ (`estimate × (1 ± (1 − confidence) × 0.5)`) даёт эталонные 7–11 точно: 9×0.81=7.29→7, 9×1.19=10.71→11.
2. **[DECISION #6] отклонён — критично** — архитектор прочитал FR-11 как runtime-лог пересчётов («Пользователь отредактировал сигнал»). По ТЗ MOX это журнал AI-процесса разработки (5–7 курируемых чекпоинтов: промпт → результат → решение → проверка). Runtime-лог означал бы автоматический незачёт.
3. **Шаг 3 движка исправлен** — архитектор округлял оценку по каждой локации до суммирования; по ТЗ суммируются неокруглённые значения, округляется только итог (на seed-данных совпадает случайно: 2+3+2+2=9, но в общем случае — ошибка).
4. **[DECISION #4] изменён** — вместо регистрозависимых локаций («Сарай» ≠ «сарай») выбрана нормализация (trim + без учёта регистра) для группировки: иначе опечатка фермера незаметно завышает оценку, разбивая одну территорию на две.
Проверка: ручной пересчёт эталонного примера из §7 на бумаге (8.94 → ≈9; 62%; 7–11); все правки внесены в PLAN.md с пометками [CORRECTED/AMENDED by orchestrator].

**🚦 GATE 1 пройден** — план представлен, возражений не поступило в отведённое окно; политика «молчание = согласие» была объявлена заранее. Все [DECISION] закрыты в пользу спецификации.

## CP-2 — Слайс 1: движок оценки + тесты (Phase 2, S1)
Задача, которую я поставил AI: workflow из трёх агентов — (1) имплементер: чистая функция `estimate()` + `recommendations.ts` + `validate.ts` + `defaults.ts` + полный Vitest-набор с закреплённым эталоном §7; (2) два параллельных адверсариальных проверяющих: один НЕЗАВИСИМО пересчитывает seed-математику из спецификации до чтения кода и сверяет с ожиданиями тестов, второй проверяет чистоту функции, замороженный API, генерацию объяснений из вычислений.
Что предложил AI: движок зелёный с 32 тестами (эталон: 8.94 → 9 кроликов, 62% «средняя», диапазон 7–11 — совпал с ручным пересчётом). Проверяющие нашли 4 недочёта (blocking — 0): (а) в причине рекомендации про ямки сумма `count` подписана как «сигналов» (для seed: 1 сигнал, но «сигналов: 2»); (б) захардкоженные '0' в объяснениях факторов вместо интерполяции; (в) сломанные русские множественные формы («2 локация»); (г) вес со слайдера мог утечь в объяснение как 0.30000000000000004.
Моё решение: все 4 исправил сам (точечные правки — делегировать 4 однострочника дороже, чем сделать): добавил `format.ts` (pluralRu + round2), переписал объяснения факторов на интерполяцию, «новых ямок: N» вместо «сигналов: N», `toFixed(1)` для средней интенсивности.
Проверка: `npm test` — 32/32 зелёные, `npm run build` — exit 0; независимый пересчёт математики верификатором сошёлся с тестовыми ожиданиями до внесения правок (правки не трогали числа — только строки объяснений).

## CP-4 — Слайс 3: интерактивность + найденные баги (Phase 2, S3)
Задача, которую я поставил AI (агент-имплементер): таблица сигналов (добавить/править/удалить/выключить), панель параметров (8 слайдеров + сброс), JSON импорт/экспорт через движковый `validateImport`, 3 сценария-пресета с закреплёнными тестами свойств («Тихое утро» → уверенность < 40; «Нашествие» → ≥ 10 кроликов + алерт), кнопка «Сбросить к исходным данным», localStorage с try/catch, вкладка AI Worklog с карточками из `src/data/worklog.ts`.
Что предложил AI: всё реализовано (35 тестов, сборка зелёная). Агент один раз упал на середине (обрыв соединения API) — возобновил его с того же транскрипта, дописал с места обрыва.
Моё решение: принял, но моя интерактивная проверка в живом предпросмотре нашла **3 реальных бага**, которые тесты не ловят:
1. **Горизонтальный скролл на 390px** (scrollWidth 612): grid-элементы с `min-width: auto` растягивались под внутреннюю ширину таблицы сигналов. Исправил: `min-width: 0` для детей грид-рядов + `min-width: 560px` на таблице со скроллом внутри собственного контейнера.
2. **Неверное объяснение в hero-карте**: «почему?» показывал `confidence/100` (0.62) под подписью «overlap discount» (реальный параметр — 0.6). Числовое совпадение 0.62 ≈ 0.6 маскировало подмену — ровно та ошибка, против которой построен критерий «объяснения из вычислений». Переписал: теперь выводится реальная сумма по локациям («У забора» ≈ 2.88 + «Сарай» ≈ 2.28 + … = 8.94 → 9) плюс англицизм ушёл.
3. **Жаргон в ошибке импорта**: «Невозможно распарсить JSON» → «Не удалось прочитать JSON — проверьте синтаксис (кавычки, запятые, скобки)».
Проверка: 35/35 тестов, build exit 0; интерактивный прогон: правка count 5→50 → ≈25 + алерт «нашествие»; выключение сигнала → ≈7 (5–9) — сошлось с ручным пересчётом (8.94−1.8·10−…); слайдер веса 1.2→2 → ≈11 (ручной пересчёт 10.86 ✓); пресеты (≈43 / ≈1 + «ненадёжна»); битый JSON → русская ошибка без падения; импорт «Курятник» применился; сброс вернул ≈9; вкладка Worklog рендерит 6 чекпоинтов; 390px без переполнения.

## CP-5 — Phase 3: адверсариальное ревью и триаж (Gate 2)
Задача, которую я поставил AI: workflow из 3 параллельных ревьюеров с разными линзами — (1) FR-1…12 + независимый пересчёт математики; (2) объяснимость / «в UI нет математики» / русская копия / темы worklog; (3) edge-cases + мусорный ввод + скан на секреты; затем фикс-агент по blocking/should-fix.
Что предложил AI: 10 находок (2 blocking, 6 should-fix, 2 optional). По делу: английские «AI Worklog»/«JSON» в шапке; ConfidenceGauge локально пересчитывал ярлык уверенности (порог 40/70 захардкожен в UI — нарушение «UI без математики»); форма позволяла сигнал с пустой локацией (молча выпадал из расчёта); риск коллизий ID при импорте внешних ID; вклады не ограничены топ-10 из PLAN. Фикс-агент применил всё, тесты 35/35.
Моё решение — триаж с двумя откатами решений фикс-агента:
1. **Откат: имя вкладки.** Фикс-агент переименовал «AI Worklog» → «Журнал разработки» ради «русской копии». Вернул «AI Worklog»: это имя обязательного артефакта из самого ТЗ MOX (§2.5 показывает таб «Worklog»), по нему проверяющий ищет вкладку. «Данные JSON» оставил.
2. **Откат: фикция в журнале.** Ревьюеры пометили заглушку чекпоинта №6 как blocking, и фикс-агент «закрыл» её выдуманным отчётом о финальной проверке (скриншоты 768×1024, hard-refresh localStorage, «10/10 smoke») — ничего из этого не выполнялось. Это анти-паттерн «журнал задним числом». Переписал чекпоинт честно: теперь он описывает само ревью Phase 3; финальная проверка появится в журнале, когда реально пройдёт.
3. window.alert из фикса заменил на встроенную ошибку формы («Укажите локацию…»).
Проверка: `git diff` тестовых файлов пуст (не ослаблены); npm test 35/35; build exit 0; живая проверка — hero ≈9, ярлык «средняя» теперь приходит из движка, вкладки «Данные»/«AI Worklog», 390px чисто.

**🚦 GATE 2 пройден** — blocking-находок не осталось; отчёт ревью и триаж зафиксированы выше.
